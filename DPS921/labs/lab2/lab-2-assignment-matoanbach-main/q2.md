# Lab 2 - Question 2 (MPI Parallel Implementation)

## Goal

Parallelize the sequential discretization algorithm using MPI collective communication while preserving the same core algorithm.

## MPI Operations Used

| MPI operation | Purpose |
|---|---|
| `MPI_Scatter` | Distributes equal-sized input chunks from rank 0 |
| Local computation | Each rank discretizes and counts its assigned data only |
| `MPI_Reduce` | Sums local zero counts into a global zero count |
| `MPI_Gather` | Collects local timing values for reporting |
| `MPI_Wtime` | Measures MPI wall-clock runtime |

## Process Responsibilities

- Rank 0 allocates and initializes the full input array.
- All ranks receive one chunk using `MPI_Scatter`.
- All ranks compute only on their local chunk.
- All ranks participate in `MPI_Reduce` to produce the final global count.
- Rank 0 prints final counts and timing information.

## Discussion

- The lowest measured Q2 wall time was at 16 processes: 0.153558 seconds.
- 8 processes was slower than 4 processes, and 32 processes was slower than 16 processes.
- Communication and synchronization prevent ideal scaling.
- On the 8-core test machine, 16 and 32 MPI processes oversubscribe the CPU.
