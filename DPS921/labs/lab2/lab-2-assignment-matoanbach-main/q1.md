# Lab 2 - Question 1 (Sequential Baseline)

## Goal

Measure the sequential runtime for the array discretization program and use it as the baseline for MPI speedup calculations.

## Methodology

- Program: `q2sequential.cpp`
- Compiler: `clang++`
- Build command: `clang++ -O2 -std=c++17 q2sequential.cpp -o build/q2sequential`
- Timing method: `std::chrono::steady_clock`, reported in seconds
- Runs: 3 per input size, average reported

The sequential and MPI versions use the same deterministic random seed and the same double-precision expression inside the discretization kernel.

## Discussion

- Runtime increases with input size because the algorithm performs initialization, discretization, and counting for every element.
- The sequential baseline is required for speedup and efficiency calculations.
- The baseline also highlights the limit of parallel improvement because MPI adds communication and synchronization overhead.
